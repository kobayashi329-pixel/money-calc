## 1. テクニカルSEO（基盤）

- ✅ HTTPS / wwwへ308正規化（apex→www）
- ✅ `lang="ja"`（layout.tsx）
- ✅ `sitemap.xml`（レジストリ駆動・`lastModified`付き）
- ✅ `robots.txt`（全許可＋sitemap明示）
- ✅ 各ページ `canonical`
- ✅ SSG（全ページ静的＝高速・クロール容易）
- ⚠️🟡 `robots.ts` に `host` 指定なし／不要URLの除外ルールなし（現状は全許可でOK、将来パラメータURLが増えたら要検討）
- ❌🟡 **og:image（OGP画像）** 未設定。SNS/検索のサムネ無し→CTR損。既定画像＋計算機ごとの動的OG画像（`opengraph-image.tsx`）
- ❌🟢 `twitter:card`（summary_large_image）未設定
- ❌🟢 favicon / `apple-touch-icon` / `manifest.webmanifest`（PWA的な信頼シグナル）の整備状況を確認
- ❌🟢 404ページのカスタム化（内部リンク付き`not-found.tsx`）でクロール離脱防止
- 🟢 多言語不要（日本語のみ）→ hreflang不要

## 2. オンページSEO（各ページ）

- ✅ title（テンプレート `%s | お金の計算機`）・description 各ページ個別
- ✅ H1 各ページ1つ・キーワード含む
- ✅ 見出し階層（H1→H2→H3）概ね適切
- ⚠️🔴 **トップ以外のtitleの最適化余地**：各計算機の検索語（例「年収手取り 計算」「ふるさと納税 上限 シミュレーション」）を意識した語順・数値訴求（「2025年」「無料」）の再点検
- ⚠️🟡 description の文字数（全角60〜80字）と検索語の自然な含有を全ページ再点検
- ⚠️🟡 **カテゴリページが薄い**（`app/c/[slug]`：H1＋1行＋カードのみ）。各カテゴリ200〜400字の独自解説＋「このカテゴリで解決できること」を追加
- ❌🟡 各計算機ページに**更新日（最終確認日）**の可視表示（鮮度シグナル＋信頼性）
- 🟢 画像が少なくalt問題はほぼ無し（チャートはSVG＋aria-label済み）

# AdSense 広告の表示形式・配置（承認後）

## 広告タイプ
| タイプ | 用途 |
| --- | --- |
| ディスプレイ広告 | 記事末尾・計算結果の下 |
| 記事内広告（インアーティクル） | ガイド記事のH2の合間に1〜2個 |
| インフィード広告 | /guide一覧・トップのカード間 |
| 関連コンテンツ（Multiplex） | 記事末尾の関連リンク前 |
| アンカー広告 | モバイル画面下に固定 |
| 全画面（Vignette） | **非推奨**（UX/審査リスク） |

## 配置方針（収益×UX）
- 計算機ページ：**「入力〜計算結果の間」には置かない**。広告は計算結果の下／解説MDX内・下。
- ガイド記事：記事内1〜2＋末尾＋Multiplex。コンテンツが厚く相性良。
- モバイル：アンカー広告は可。ポップアップ系は避ける。
- 最初は**手動配置（位置を制御）**、慣れたら自動広告を限定ON。

## 実装メモ
- 広告コードを共通`<AdSlot>`コンポーネント化し記事/計算機ページへ差し込む（pub-ID＝`ADSENSE_CLIENT`設置済み）。
- **CLS対策**：各枠に`min-height`予約。
- ポリシー：広告とコンテンツを明確に区別／計算機操作を妨げない／1ページの広告過多回避。
- 同意メッセージ（CMP）：EEA向けにGoogle CMPを設定（自動表示・追加実装不要）。
